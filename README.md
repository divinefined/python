## 🎮 Проекты

### 1. 🐾 **Animal - Tamagotchi**

**Описание:** Виртуальная игра-симуляция по уходу за цифровым питомцем.

**Технологии:** Python, многопоточность (`threading`), синхронизация потоков (`RLock`)

**Возможности:**
- Питание питомца с валидацией входных данных
- Развлечение питомца (влияет на счастье и сытость)
- Мониторинг сытости и счастья в реальном времени
- Асинхронный жизненный цикл в отдельном потоке
- Потокобезопасность с использованием `RLock` (переентрантная блокировка)
- Обработка некорректного ввода (не-числовые значения, нулевые/отрицательные)

**Ключевые концепции:**
- **RLock вместо Lock** — позволяет одному потоку повторно захватывать замок
- **Race condition** — защита `is_alive` с блокировкой
- **Deadlock prevention** — разделение `_check_status()` (без блокировки) и `check_status()` (с блокировкой)

**Запуск:**
```bash
cd Animal
python animal.py
```

**Пример использования:**
```
Введите команду (feed, play): feed
Введите количество еды: 30
Барсик накормлен(а)
Сытость  100
Радость:  100

Введите команду (feed, play): play
Введите количество времени: 20
Барсик поиграл(а)
Сытость  80
Радость:  100
```

**Статус:** ✅ **Работает** (исправлены deadlock и валидация)

### 2. 📝 **Modify - Custom List**

**Описание:** Кастомный класс списка с перегрузкой операторов.

**Технологии:** Python OOP, перегрузка операторов (`__sub__`, `__truediv__`, `__str__`)

**Возможности:**
- Вычитание списков (разность множеств)
- Вычитание скаляра из всех элементов
- Деление всех элементов на скаляр
- Кастомное строковое представление
- Проверка деления на ноль

**Ключевые концепции:**
- **Наследование** встроенного типа `list`
- **Перегрузка операторов** для пользовательской логики
- **Полиморфизм** — `__sub__` работает по-разному в зависимости от типа параметра

**Запуск:**
```bash
cd Modify
python modify.py
```

**Пример:**
```python
l1 = My_list(1, 2, 3)
l2 = My_list(3, 4, 5)
print(l1 - l2)   # My_list contains: [1, 2]
print(l1 - 1)    # My_list contains: [0, 1, 2]
print(l1 / 5)    # My_list contains: [0.2, 0.4, 0.6]
```

**Вывод при запуске:**
```
My_list contains: [0.2, 0.4, 0.6]
My_list contains: [4, 5]
My_list contains: [1, 2]
```

**Статус:** ✅ **Работает** (протестировано)

### 3. 🌍 **Sun_system - Solar System Simulator**

**Описание:** Интерактивная визуализация Солнечной системы с движением планет.

**Технологии:** Pygame, математика (тригонометрия), физика орбит

**Возможности:**
- Визуализация 8 планет с реальными орбитальными периодами
- Орбитальные движения с масштабированием времени (`× 0.05` для видимости)
- Интерактивное наведение мышкой — информация о планете
- Счетчик дней (на основе орбиты Земли = 365.2 дня)
- Центральное Солнце с правильным масштабом

**Ключевые концепции:**
- **Полярные координаты** — угол и расстояние от центра
- **Физика орбит:** `angle += (dt / period) * 2π`
- **Координаты планеты:** `x = r × cos(θ) + center_x`, `y = r × sin(θ) + center_y`
- **Детекция наведения** через вычисление расстояния до планеты
- **Использование абсолютных путей** для загрузки изображений

**Требования:**
- Python 3.6+
- `pygame`
- PNG изображения планет (mercury.png, venus.png, earth.png, mars.png, jupiter.png, saturn.png, uranus.png, neptune.png)

**Запуск:**
```bash

```

### 4. 📐 **Triangulate - Triangle Analyzer**

**Описание:** Анализ всех возможных треугольников, поиск с минимальной и максимальной площадью.

**Технологии:** Computational Geometry, регулярные выражения, комбинаторика

**Возможности:**
- Чтение точек из текстового файла в формате `[x,y][x,y]...`
- Генерация всех возможных комбинаций 3 точек ($ \binom{n}{3} $ треугольников)
- Вычисление площади по формуле Герона
- Фильтрация вырожденных треугольников (площадь = 0)
- Поиск минимального и максимального треугольника по площади
- Сравнение треугольников через перегрузку операторов

**Ключевые концепции:**
- **Формула Герона:** $S = \sqrt{s(s-a)(s-b)(s-c)}$, где $s = \frac{a+b+c}{2}$
- **Регулярные выражения** — `re.findall(r'\[([-\d.]+),\s*([-\d.]+)\]', data)`
- **Перегрузка операторов сравнения** (`__lt__`, `__le__`, `__eq__`, `__gt__`, `__ge__`, `__ne__`)
- **Комбинаторика** — 3 вложенных цикла: $O(n^3)$ сложность
- **Обработка граничных случаев** — коллинеарные точки, нулевая площадь

**Запуск:**
```bash
cd Triangulate/triangules
python tri.py
```

**Пример вывода:**
```
Найдено точек: 100
Точек после преобразования: 100
Треугольник с минимальной площадью: Triangle(...) (площадь: 2.37e-06)
Треугольник с максимальной площадью: Triangle(...) (площадь: 1180.0)
```

**Статус:** ✅ **Работает** (протестировано с 100 точками)

### 5. 🎮 **Zeros - Tic-Tac-Toe**

**Описание:** Классическая игра Крестики-Нолики с графическим интерфейсом на Pygame.

**Технологии:** Pygame, игровая логика, GUI

**Возможности:**
- Графический интерфейс 3×3 сетка
- Два игрока (X и O) с чередованием ходов
- Проверка победителя (3 в ряд по горизонтали, вертикали, диагонали)
- Проверка ничьей (все клетки заполнены)
- Кнопка перезагрузки для новой игры
- Интерактивное управление (клик мышкой)
- Визуальная обратная связь (статус игры на экране)

**Ключевые концепции:**
- **Game State** — матрица 3×3 для хранения ходов
- **Win Condition Check** — проверка 8 возможных линий победы
- **Event Handling** — обработка кликов мышки в Pygame
- **Button Collision Detection** — определение клика по кнопке
- **Game Loop** — основной цикл обновления и рендеринга

**Требования:**
- Python 3.6+
- `pygame`

**Запуск:**
```bash
pip install pygame
cd Zeros
python zeros.py
```

**Правила:**
1. X начинает первым
2. Игроки ходят по очереди
3. Первый, кто выставит 3 в ряд — победитель
4. Если все 9 клеток заполнены без победителя — ничья

**Управление:**
- Клик на клетку — выполнить ход
- Кнопка "Перезапуск" — новая игра

**Статус:** ✅ **Синтаксис верен** (требует Pygame, протестировано логика)
