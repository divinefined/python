# Крестики-Нолики (Tic-Tac-Toe)

Классическая игра Крестики-Нолики с графическим интерфейсом на базе Pygame.

## Описание

Это полнофункциональная реализация классической игры **Крестики-Нолики** (3x3 доска) с:
- **Графическим интерфейсом** на Pygame
- **Двумя игроками** (X и O чередуются)
- **Проверкой победы** по горизонталям, вертикалям и диагоналям
- **Проверкой ничьей** (все клетки заполнены без победителя)
- **Кнопкой перезагрузки** для повторной игры
- **Интерактивным управлением** — клик мышкой по клетке

## Требования

- Python 3.6+
- **pygame** — библиотека для графики

## Правила игры

1. **Доска**: Сетка 3×3 (всего 9 клеток)
2. **Игроки**: Два игрока (X и O)
3. **Ход**: Доступный ход состоит в нажатии на пустую клетку
4. **Очередность**: X начинает первым, затем O, и так по очереди
5. **Победа**: Три одинаковых символа подряд (горизонталь, вертикаль или диагональ)
6. **Ничья**: Все 9 клеток заполнены, но нет победителя
7. **Перезагрузка**: Нажмите кнопку "Перезапуск" для новой игры

## Управление

- **Клик мышки** на клетке доски — сделать ход
- **Кнопка "Перезапуск"** в нижнем правом углу — начать новую игру

## Визуальные элементы

- **Черная сетка** — разделение доски на 9 клеток
- **Черные крестики (X)** — ходы первого игрока
- **Черные окружности (O)** — ходы второго игрока
- **Голубая кнопка** — кнопка перезагрузки
- **Сообщение о результате** — отображается при победе или ничьей

## Структура кода

### Класс `Board`

Управляет состоянием игровой доски.

**Атрибуты:**
- `board` — матрица 3×3 с состоянием клеток
- `current_player` — текущий игрок ("X" или "O")
- `winner` — победитель (если есть)
- `draw_flag` — флаг ничьей

**Методы:**
- `draw(screen)` — отрисовать доску
- `mark_cell(row, col)` — сделать ход на клетку [row][col]
- `check_winner()` — проверить, есть ли победитель
- `check_draw()` — проверить, ничья ли
- `reset()` — очистить доску и начать заново

### Класс `Game`

Управляет игровым процессом и графическим интерфейсом.

**Методы:**
- `draw_text(text, color, position)` — рисовать текст на экране
- `draw_reset_button()` — рисовать кнопку перезагрузки
- `handle_reset_button_click()` — обработать клик по кнопке

## Пример игровой сессии

1. Игрок X кликает на центр — X появляется в центре
2. Игрок O кликает на угол — O появляется в углу
3. Так продолжается, пока:
   - Кто-то не выставит три символа подряд (победа), или
   - Все 9 клеток не заполнятся (ничья)

## Алгоритм проверки победы

Проверяются все возможные комбинации:

**Горизонтали:**
```
Row 0: [0,0] [0,1] [0,2]
Row 1: [1,0] [1,1] [1,2]
Row 2: [2,0] [2,1] [2,2]
```

**Вертикали:**
```
Col 0: [0,0] [1,0] [2,0]
Col 1: [0,1] [1,1] [2,1]
Col 2: [0,2] [1,2] [2,2]
```

**Диагонали:**
```
Main: [0,0] [1,1] [2,2]
Anti: [0,2] [1,1] [2,0]
```

## Выигрышные стратегии

1. **Центр** — центральная клетка имеет наибольшее значение
2. **Углы** — углы имеют больше связей, чем стороны
3. **Блокировка** — всегда блокируйте ход противника, если он один ход до победы

## Известные особенности

- При ничьей на экран не выводится сообщение (можно улучшить)
- Невозможно отменить ход после его совершения
- Кнопка перезагрузки может быть нажата в любой момент
